# 记录想法与经验

## 描述

本技能用于记录用户的想法和经验，支持分类和任务关联功能，确保内容能够结构化存储并便于后续检索和汇总。

## 使用场景 (Boundary)

### 正向触发 (Use this skill when)
- 用户提出："我有个想法"
- 用户提出："记录想法"
- 用户提出："记录经验"

### 负向触发 (Do NOT use this skill when)
- 用户明确说"**记录日志**"（应使用 generating-reports 技能或直接操作日志文件）
- 用户只是在咨询通用的问题而非请求记录想法或经验
- 用户只是在讨论想法或经验而没有明确要求记录

## 输入输出定义 (Structured I/O)

- **Input**:
  - `内容类型`: 想法 或 经验
  - `内容文本`: 用户想要记录的想法或经验文本
  - `分类`: 想法或经验的分类
  - `关联任务`: 关联的任务文件名（可选）
- **Output**:
  - `记录结果`: 确认内容已记录到指定文件

## 执行步骤 (Steps)

1. **识别内容类型**: 判断用户是要记录"想法"还是"经验"
2. **收集分类信息**:
   - 如果是**想法**，询问分类（项目、技术、产品、学习、其他），默认"其他"
   - 如果是**经验**，询问分类（运维经验、大模型应用经验、AI Agent使用经验、编程经验、其他）
3. **获取任务列表**: 使用 Glob 工具获取 `tasks/` 目录下的任务文件名（**排除 `tasks/done/` 目录及其子目录**），排除 templates 目录
4. **选择关联任务**: 列出任务文件名列表供用户选择，提供"不关联"选项
5. **确认内容文本**: 让用户确认要记录的内容
6. **创建目录和文件**:
   - 如果是想法，存储到 `ideas/` 目录
   - 如果是经验，存储到 `experience/` 目录
   - 如果对应分类文件不存在，则创建该文件
7. **追加记录**: 使用 Edit 工具将内容按指定格式追加到分类文件末尾
8. **验证结果**: 读取文件最后几行验证记录是否成功
9. **反馈用户**: 向用户确认记录成功及文件位置

## 记录格式

### 想法记录格式
```markdown
## [YYYY-MM-DD HH:MM:SS]
想法：[想法内容]
关联任务：[任务文件名 或 "无"]
```

### 经验记录格式
```markdown
## [YYYY-MM-DD HH:MM:SS]
经验：[经验内容]
关联任务：[任务文件名 或 "无"]
```

## 文件命名规则

- **想法文件**：存储在 `ideas/` 目录，按分类命名 `[分类名].md`
  - 例如：`ideas/技术.md`、`ideas/产品.md`、`ideas/其他.md`
  
- **经验文件**：存储在 `experience/` 目录，按分类命名 `[分类名].md`
  - 例如：`experience/运维经验.md`、`experience/大模型应用经验.md`、`experience/AI Agent使用经验.md`、`experience/编程经验.md`

## 失败策略 (On Failure)

- **内容类型不明确**: 若无法判断是想法还是经验，询问用户确认
- **分类不明确**: 若用户无法确定分类，想法默认使用"其他"，经验默认使用"编程经验"
- **任务文件获取失败**: 若无法获取任务文件列表，跳过任务关联步骤，直接记录内容
- **文件写入失败**: 若无法写入文件，向用户报告错误并尝试将内容显示给用户

## 示例 (Examples)

### 示例1：记录技术想法，关联任务

**输入**
> "我有个关于askdb项目的想法，记录一下"

**执行流程**
1. 识别为"想法"
2. 询问分类：技术
3. 列出任务文件供选择，用户选择"askdb项目"
4. 确认想法内容
5. 记录到 `ideas/技术.md`

### 示例2：记录运维经验，不关联任务

**输入**
> "记录一个运维经验"

**执行流程**
1. 识别为"经验"
2. 询问分类：运维经验
3. 列出任务文件供选择，用户选择"不关联"
4. 确认经验内容
5. 记录到 `experience/运维经验.md`

### 示例3：记录大模型应用经验

**输入**
> "我发现了使用大模型的一个技巧，记下来"

**执行流程**
1. 识别为"经验"
2. 询问分类：大模型应用经验
3. 列出任务文件供选择
4. 确认经验内容
5. 记录到 `experience/大模型应用经验.md`
