# 技能创建与迭代指南 (learn-skills)

## 描述

本技能是关于“如何编写技能”的技能。它将 TRAE 的官方 Skill 设计标准（边界、结构化、指令化、失败策略、单一职责）转化为可执行的开发流程。

**参考模板文件**：`.trae/skills/learn-skills/templates/SKILL-template.md`

## 使用场景
### 正向触发 (Use this skill when)
- 用户提出：“帮我写一个处理 XXX 任务的技能”。
- 用户需要优化现有技能的触发命中率或执行稳定性。
- 需要将复杂的 Prompt 转化为长期复用的技能模块。

### 负向触发 (Do NOT use this skill when)
- 用户只是在咨询通用的 IDE 设置或规则（Rules）。
- 任务仅为一次性的临时提问，不需要工程化维护。

## 输入输出定义 (Structured I/O)
- **Input**: 目标任务描述、现有代码上下文、已知的失败案例。
- **Output**: 符合规范的 `SKILL.md` 文件内容及建议的目录结构,。

## 执行步骤 (Steps)
1. **基线分析**：先模拟无技能执行，识别模型容易产生歧义或走偏的具体环节。
2. **定义边界**：明确该技能的 `When`（正向触发）和 `When NOT to use`（负向触发）条件。
3. **结构化 I/O**：使用函数签名风格定义技能的输入（Input）和输出（Output）。
4. **指令化编写**：使用**指令式动词**（如 Validate, Checkout, Collect）编写具体的执行步骤。
5. **制定失败策略**：定义当环境不匹配、参数缺失或执行异常时的回退方案。
6. **单一职责检查**：确保该技能只负责一个核心动作。若过于复杂，建议拆分为多个子技能。
7. **同步项目资产**：**必须更新项目根目录下的 `AGENTS.md`**，在其中添加或修正新技能的调用逻辑；同时**更新 `README.md`**，同步修改其中的系统架构视图和技能功能说明部分。

## 失败策略 (On Failure)
- **目标过于模糊**：若无法推导出明确的触发条件，需暂停并引导用户提供具体的“失败场景”作为评测依据。
- **路径冲突**：若技能涉及文件操作，使用与操作系统兼容的路径格式（Windows 使用反斜杠或原始字符串，Unix 使用正斜杠）。

## 示例 (Example)
### 输入
“帮我写一个自动清理项目临时文件的技能。”

### 输出建议
- **Name**: `cleaning-temp-files`
- **Description**: 用于识别并清理项目中的 log、tmp 等临时文件。
- **Steps**: 1. 扫描指定目录；2. 匹配后缀；3. 询问确认；4. 执行删除。
- **Doc Updates**: 更新 `AGENTS.md` 触发逻辑与 `README.md` 的技能清单。
