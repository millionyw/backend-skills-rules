# 报告生成技能

## 描述

此技能用于生成日报（Daily Log）和周报（Weekly Report），要求 AI 必须通过上下文感知，基于实际日志和任务数据生成有价值的总结与计划。

## 使用场景

### 正向触发 (When to use)

- 用户请求生成日报。
- 用户请求生成周报。
- 周末进行周度回顾。
- 用户需要在 `reports/` 文件夹下创建报告文档。

### 负向触发 (When NOT to use)

- 用户只是查询现有报告内容，不涉及文件创建。
- 用户请求更新任务状态（应使用 tracking-progress 技能）。
- 用户请求创建新任务（应使用 managing-tasks 技能）。

## 指令

### 日报生成

#### 步骤一：收集上下文

1. 确定今日日期
2. 读取今日日志模板或已有日志
3. **日志文件必须位于 `logs/YYYY-MM/YYYY-MM-DD.md`**
4. 如果无日志，提示用户先记录

#### 步骤二：结构化输出

按照日报模板生成内容：

```markdown
# 工作日志 - YYYY-MM-DD

## 今日总结

### 完成事项 ✅
- [ ] 事项1

### 进行中事项 🔄
- [ ] 事项1

### 待处理事项 ⏳
- [ ] 事项1

## 任务进度

| 任务ID | 任务名称 | 今日进展 | 完成度 |
|--------|----------|----------|--------|

## 下周计划 / 明日计划

- [ ] 计划1

## 备注

```

#### 周五特殊处理

**判断逻辑**：如果今天是周五（本周第 5 天），则：
- 将"明日计划"改为"**下周计划**"
- 计划内容应包含：
  1. 本周未完成事项的延续
  2. 下周重点任务安排
  3. 需要跟进的问题

**周五日志模板**：

```markdown
## 下周计划

### 重点任务

- [ ] 任务1

### 待跟进事项

- [ ] 事项1
```

#### 步骤三：保存文件

- 文件保存路径：`logs/YYYY-MM/daily-YYYY-MM-DD.md`
- 如果是首次创建，使用 `logs/templates/daily-log-template.md` 作为模板

### 周报生成

#### 步骤一：收集上下文（必须）

1. 确定周次（YYYY-Wxx）
2. **读取本周所有日志文件：`logs/YYYY-MM/YYYY-MM-DD.md`**
3. **只读取 `tasks/` 目录下的任务文件（不包含 `tasks/done/` 目录）**
4. 如果有上周周报，读取以保持连续性

#### 步骤二：分析数据

对收集的数据进行分析：

1. **完成事项统计**：统计本周完成的任务数
2. **进度变化**：分析各任务的完成度变化
3. **问题汇总**：提取本周遇到的问题
4. **时间分配**：分析时间投入情况

#### 步骤三：生成周报

按照周报模板生成内容，确保包含：

1. **本周概览**：关键成果、数据统计
2. **任务进度**：已完成/进行中/新增任务
3. **问题与风险**：本周问题、风险预警
4. **下周计划**：优先级事项、预期成果

#### 步骤四：保存文件

- 文件保存路径：`reports/YYYY-Wxx.md`
- 使用 `reports/templates/weekly-report-template.md` 作为模板

## 约束

- **必须读取上下文**：生成报告前必须读取相关日志和任务文件
- **日志路径约束**：日志文件必须位于 `logs/YYYY-MM/` 目录下，禁止放在 `logs/` 根目录
- **禁止虚构**：所有内容必须基于实际数据，不能编造
- **结构化输出**：必须使用模板格式，确保一致性
- **保存文件**：生成后必须保存到指定路径
- **如果日志缺失**：提示用户先补全日志，无法生成完整报告
- **不读取已完成任务**：只读取 `tasks/` 目录下的任务文件，不读取 `tasks/done/` 目录（已完成任务已归档到对应周次目录）
