# 进度对齐技能

## 描述

此技能用于将每日工作日志中的实际执行情况与计划任务进行比对，自动更新任务状态和完成度，确保任务进度与实际进展保持一致。任务完成时自动归档到 `tasks/done/YYYY-MM` 目录。

## 使用场景

### 正向触发 (When to use)

- 用户请求更新任务进度。
- 用户希望同步日志与任务状态。
- 定期进行进度回顾。
- 需要在 `tasks/` 文件夹下更新任务状态。
- 需要在 `logs/` 文件夹下记录或比对执行情况。

### 负向触发 (When NOT to use)

- 用户只是查询某个任务的状态，不涉及文件修改。
- 用户要求生成报告（应使用 generating-reports 技能）。
- 用户仅要求查看日志内容而不任务更新。

## 指令

### 步骤一：收集数据

1. **读取任务文件**
   - 扫描 `tasks/` 目录下所有任务文件（**不包含** `tasks/done/` 目录）
   - 列出所有进行中（未完成）的任务

2. **读取日志文件**
   - **日志文件必须位于 `logs/YYYY-MM/YYYY-MM-DD.md`**
   - 重点关注最近一周的日志
   - 提取每条日志中的任务进度更新

### 步骤二：分析对比

对每个进行中的任务进行分析：

1. 检查日志中是否有该任务的进展记录
2. 分析完成度变化：
   - 有进展记录：根据记录更新完成度
   - 无进展记录：检查是否过期或存在阻塞
3. 判断状态变更：
   - 完成度 100% → 状态变为 ✅ 已完成
   - 超过截止日期且完成度 < 100% → 状态变为 🔴 Red
   - 超过原定里程碑日期 → 状态变为 🟡 Yellow

### 步骤三：状态更新

根据分析结果更新任务文件：

1. 更新任务的当前完成度
2. 更新任务状态标识
3. 记录上次更新日期
4. **任务归档**：当任务状态变为"已完成"（完成度 100%）时：
   - 提取当前日期的年月（YYYY-MM 格式）
   - 创建目标目录 `tasks/done/YYYY-MM/`（如不存在）
   - 使用 Move 工具将任务文件移动到 `tasks/done/YYYY-MM/{原文件名}`
   - 更新日志中记录归档操作

### 步骤四：输出报告

生成进度对齐报告，格式如下：

```markdown
## 进度对齐报告

### 状态汇总

| 状态 | 数量 |
|------|------|
| 🟢 正常 | X |
| 🟡 有风险 | X |
| 🔴 阻塞 | X |

### 任务更新详情

| 任务ID | 任务名称 | 原完成度 | 新完成度 | 状态变化 |
|--------|----------|----------|----------|----------|
| T001 | 任务名 | 50% | 70% | 🟢 → 🟢 |
```

### 步骤五：风险预警

识别并报告存在风险的任务：
- 超过截止日期的任务
- 连续多日无进展的任务
- 完成度远低于预期的任务

## 约束

- **日志路径约束**：日志文件必须位于 `logs/YYYY-MM/YYYY-MM-DD.md`，禁止放在 `logs/` 根目录
- 每次更新必须读取相关日志文件
- 状态变更是基于日志证据，不能主观臆断
- 更新后必须记录变更历史
- 如果日志缺失，提示用户先补全日志
- **归档约束**：任务完成（完成度 100%）时必须归档到 `tasks/done/YYYY-MM/` 目录
- **不读取已完成任务**：进度更新时不应读取 `tasks/done/` 目录下的文件
